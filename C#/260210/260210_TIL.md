# MVP 패턴
MVP 패턴이란? 

MVP 패턴의 핵심은 **"데이터(Model)와 화면(View)을 완전히 찢어놓는 것"**입니다. 그 사이에서 중매쟁이 역할을 하는 Presenter를 두어 서로의 존재를 모르게 만든다.

- M : Model
- V : View
- P : Presenter

## 장점
1. 유지보수성
2. 교체 가능성
3. 테스트 용이성

## 잘못된 예
```csharp
using TMPro;
using UnityEngine;

public class Enemy : MonoBehaviour
{
    public int hp = 100;

    public TextMeshPro hpText;

    public void OnDamage(int damage)
    {
        hp -= damage;
        hpText.text = "HP" + hp.ToString();

        if(hp <= 0)
        {
            Debug.Log("Enemy 죽음..");
        }
    }
}
```

## ✅ MVP를 적용한 구조
각자의 역할에 충실하도록 역할을 분리한다.

- Model: "나는 HP만 관리해. 바뀌면 알려줄게!"

- View: "나는 시키는 대로 글자만 써. 버튼 눌리면 신호 보낼게!"

- Presenter: "너희 둘은 서로 몰라도 돼. 내가 중간에서 연결해 줄게."
