## 과제 2. 출력결과 판단하기 1
```csharp
using System;
using System.Collections.Generic;

public class Solution
{
    public static void Main(string[] args)
    {
        void UpdateList(int[] arr)
        {
            arr[0] = 99;
        }

        int[] numArr = new int[] { 1, 2, 3 };
        UpdateList(numArr);
        Console.WriteLine(numArr[0]);
    }
}
```
int 배열로 선언된 numArr에서 3개의 원소 1,2,3이 저장되어있다. UpdateList함수는 int 배열을 받아 배열의 0번째 원소를 99로 바꿔주는 함수인데 언뜻 보면 값을 넘겨주는 것처럼 보여 값복사가 되어 numArr이 바뀌지 않는 것처럼 보일 수 도 있지만 배열은 ref로 넘겨주기 때문에 함수 안에서 값을 바꾸더라도 본 값이 바뀐다.

## 문제 3. 출력결과 판단하기 2
함수에 인자가 전달될 때 일어나는 일과 연관하여 어떤 출력 결과가 나오는지 설명하시오
```csharp
using System;
using System.Collections.Generic;

public class Solution
{
    public static void Main(string[] args)
    {
        void ReplaceList(int[] arr)
        {
            arr = new int[] { 9, 9, 9 };
        }

        int[] numArr = new int[] { 1, 2, 3 };
        ReplaceList(numArr);
        Console.WriteLine(numArr[0]);

        // 출력 결과: ?
    }
}
```
이 문제는 위의 문제와 비슷하게 보이지만 결과를 아예 다르다. 위의 문제에서는 같은 배열을 수정하고 있는 거라면 두 번째 문제는 새로운 배열을 하나 만들어서 arr이 가리키도록 만드는 것이다. 원래는 numArr과 arr이 똑같은 주소를 가리키고 있었다면 ReplaceList에서 새로운 배열을 만들고 arr이 가리키는 주소를 바꿔버린 것이다.

```csharp
void ModifyRef(ref int[] arr)
{
    arr = new int[] { 7, 7, 7 };
}

int[] numArr2 = new int[] { 10, 20, 30 };
ModifyRef(ref numArr2);
Console.WriteLine(numArr2[0]); 

// 출력 결과: ?
```

위의 두 함수는 동작이 거의 비슷하다. 하지만 함수 내에서의 로직이 다른데 첫번째 함수는 위에 문제를 풀었던 것처럼 numArr의 주소값을 가르키는 arr을 새로 만들어 똑같은 주소를 가리키게 하는 구조였다면 두 번째 문제는 numArr2를 함수에 직접 넘겨주어 numArr2 자체를 변화시키는 것이다.
