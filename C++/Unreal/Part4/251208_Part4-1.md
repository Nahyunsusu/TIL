# 언리얼 게임플레이 어빌리티 시스템 개요

## 학습 목표
- 게임플레이 어빌리티 시스템(GAS)을 구성하는 주요 개념과 핵심 구성요소의 학습
  

## 게임플레이 어빌리티 시스템(GAS) 프레임웍
- 액터가 소유하고 발동할 수 있는 어빌리티 및 액터 간의 인터랙션 기능을 제공하는 프레임웍
- RPG, 액션 어드벤처, MOBA 장르의 제작을 쉽게하기 위한 도구. 대부분의 게임 제작에 활용 가능
- GAS 프레임웍의 장점
 
  1. 유연성과 확장성  : 다양하고 복잡한 게임 제작에 대응할 수 있도록 범용적으로 설계
  2. 모듈러 시스템    : 각 기능에 대한 의존성이 최소화되도록 설계
  3. 네트워크 지원    : 네트웍 멀티플레이어 게임에서도 활용 가능하도록 설계
  4. 데이터 기반 설계 : 데이터를 기반으로 동작하도록 설계
  5. 완성도          : 포트나이트 게임 서비스를 통해 실효성 검증

- GAS 프레임웍의 단점
  
    1. 배우는 학습 비용 : 구성 요소가 많아서 학습하는 비용이 꽤 큼
    2. 오버헤드 : 작은 규모의 프로젝트에서는 복잡한 구조가 오히려 부담될 수 있음.
   
   큰 규모의 RPG 및 네트웍 멀티플레이 게임을 효율적으로 만드는데 적합함
